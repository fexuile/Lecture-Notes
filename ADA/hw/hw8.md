## 8.2

## 8.4

### (1)

顺序归并下：$l = n/k$

$l+l+(2l + l) +\dots + ((k-1)l + l) = l + 2l + \dots kl = \frac{lk(k+1)}{2}$

所以时间复杂度最坏为$O(nk)$的。

### (2)

我们考虑二分归并，两两分组，然后一直这样子归并上去，直到最后只剩下一个序列。

这样子的时间复杂度最坏为：

考虑每一个序列归并$\log k$次，每一次贡献$n/k$的长度，一共$k$个序列。

所以时间复杂度为:$O(n/k *k\log k) = O(n\log k)$。

### (3)

我们考虑这个问题可以转换为一开始给定一个长度为$n$的序列，然后其中有$k-1$个可供切割的位点，每一个节点表示一次切割将原本的序列分为两个子序列。如果其中一个子序列没有切割位点了，就作为叶子节点，两边一个作为左儿子，一个作为右儿子。

这样子我们将这个过程逆向实际上就唯一对应了一种合并方案。

对于这个决策树而言，一共有$k$个叶子节点，每一层都需要花费$O(n)$的时间来进行合并，所以最后时间复杂度至少是$\Omega(n\log k)$的。




## 8.7

### (1)

很明显$A_i$是递增的，那么$A_i < A_{i+1}$， $A_i - i < A_{i+1} - i$,$A_i - i \le A_{i+1} - (i+1)$。

我们就可以采用二分的方法：找到$A_i-i$这个序列中的$0$就可以了。

```
int check(){
	int l = 1, r = n, ans = 0;
	while(l < r) {
		int mid = (l + r) >> 1;
		if(a[mid] - mid <= 0) {
			ans = mid; l = mid + 1;
		}
		else r = mid - 1;
	}
	return ans;
}
```

最坏情况下复杂度为$\Theta(\log n)$

### (2)

我们将比较构建成一棵树，$A_i$和$i$进行比较：

- 如果$A_i = i$，那么就把当前节点作为叶子节点；
- 否则的话把下一次的比较作为右儿子，$i$作为左儿子。

这样子的情况下，一共有$n$个叶子节点，那么这个树的高度至少是$\log n$的。

所以求解的时间复杂度下限为$\Omega(\log n)$



## 8.10

我们还是按照每一次比较的结果进行分组。设第$k$小的数为$ans$，那么如果一个数比$ans$大，我们就称它为$L$组，比$ans$小则是$S$组，没有比较过的我们都放在$N$组里。

我们知道决定性比较$(x,y)$应该为$x > y, y \ge ans$，或者$x < y, y\le ans$，这样子的比较是$n-1$次的。

非决定性比较我们采用构造的方法：

- 对于$(N,N)$的组，我们将一个放在$L$组，一个放在$S$组。

- 对于$(L,N)$的组，我们将$N$的放在$S$组里；对于$(S,N)$同理，放在$L$组中。
- 如果已经赋值了的有$k-1$个在$S$组，或者$n-k$个在$L$组，我们就将剩余的放在另一组里，最后剩余一个分配$ans$。

这样子的情况下，这些比较都是非决定性比较，我们可知道：

比较次数至少是：$n-1+\min(k-1,n-k) = n - 2 + \min(k,n-k+1)$。
